## Phase 3 — Supabase Backend

**Supabase Schema (PostgreSQL)**

```sql
-- Users (no Supabase Auth — custom code-based auth)
create table users (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  code_hash text not null unique, -- bcrypt hash of 6-char code
  created_at timestamptz default now()
);

-- Sessions
create table sessions (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  is_active boolean not null default false,
  created_at timestamptz default now(),
  closed_at timestamptz
);

-- Pings (one per user per session — enforced by unique constraint + RPC)
create table pings (
  id uuid primary key default gen_random_uuid(),
  session_id uuid references sessions(id) on delete cascade,
  user_id uuid references users(id) on delete cascade,
  user_name text not null,
  status text not null default 'pending' check (status in ('pending','acknowledged','done')),
  note text,
  created_at timestamptz default now(),
  updated_at timestamptz default now(),
  unique(session_id, user_id)
);

-- Push subscriptions (Phase 7)
create table push_subscriptions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references users(id) on delete cascade,
  endpoint text not null,
  p256dh text not null,
  auth text not null,
  created_at timestamptz default now()
);

-- Events log (audit trail)
create table events (
  id uuid primary key default gen_random_uuid(),
  type text not null,
  payload jsonb,
  created_at timestamptz default now()
);
```

**Atomic ping creation RPC:**

```sql
create or replace function create_ping(
  p_session_id uuid,
  p_user_id uuid,
  p_user_name text,
  p_note text default null
) returns pings as $$
declare
  v_session sessions;
  v_ping pings;
begin
  -- Lock and verify session is active
  select * into v_session from sessions where id = p_session_id for update;
  if not found then raise exception 'Session not found'; end if;
  if not v_session.is_active then raise exception 'Session is not active'; end if;

  -- Insert (unique constraint prevents double-ping)
  insert into pings (session_id, user_id, user_name, note)
    values (p_session_id, p_user_id, p_user_name, p_note)
    returning * into v_ping;

  return v_ping;
end;
$$ language plpgsql security definer;
```

**Row Level Security:** All tables use custom JWT-less auth — use service role key on server-side functions only. Client uses anon key with restrictive RLS:

```sql
-- Passengers can only read their own pings
alter table pings enable row level security;
-- Add policies based on your custom session token approach
```

```typescript
  'mailto:admin@example.com',

  const { ping } = await req.json();
  // Fetch all admin push subscriptions and notify
  // ...
});
```

## Deployment (Netlify)

1. Push to GitHub
2. Connect repo to Netlify
3. Set build command: `npm run build`
4. Set publish directory: `dist`
5. Add environment variables in Netlify dashboard:
   - `VITE_BYPASS_INSTALL_GATE=false` (production)
   - `VITE_SUPABASE_URL` (Phase 3+)
   - `VITE_SUPABASE_ANON_KEY` (Phase 3+)
   - `VITE_VAPID_PUBLIC_KEY` (Phase 7)

Supabase secrets (`VAPID_PRIVATE_KEY`, service role key) go into **Supabase Edge Function secrets only** — never in Netlify.
